게임을 개발하고 운영하는 대학교 강의 조별 과제로 제작한 게임입니다.<br>학생이 게임 유저가 되어 게임을 모의 운영하는 만큼, 쉬는 시간에 간단하게 즐길 수 있는 게임을 기획하였습니다.<br>뒤끝 서버와 유니티를 연동하여 유저의 활동들을 수집하고 최고 점수를 저장할 수 있도록 했습니다.

## 구현 목록
1. 게임 흐름 관리
    - 정적 클래스인 GameManager를 중심으로 게임 상태, 점수, 게임 오버, 재시작 등 전반적인 게임 흐름을 관리하도록 구현.
    - 장애물 생성, 점수 관리 등 세부 로직은 책임 분리 원칙에 따라 별도의 스크립트로 분리하여 관리.
    - 게임 시작 시 GameManager 스크립트에서 ObstacleSpawner 스크립트를 활성화하여 장애물이 생성되도록 구현.
2. 플레이어 구현
    - 플레이어의 움직임, 충돌, 아이템 획득 로직을 클래스 단위로 분리하여 유지보수와 기능 확장이 용이한 구조를 설계.
3. 상속을 통한 장애물 구현
    - 장애물의 공통 속성과 동작을 가진 ObstaclePositioner 클래스를 상속하여 고정형 장애물과 이동형 장애물을 구현함으로써 코드 재사용성을 향상.
    - 코루틴을 사용하여 장애물의 생성 주기를 관리.
4. UI
    - UGUI 기반으로 메인 화면, 게임 화면, 랭킹 화면 등 전체 UI를 구성.
    - MVP 패턴을 적용하여 데이터와 구현부를 분리하고 UI의 확장성과 유지보수성을 향상.
    - DoTween을 활용하여 UI 애니메이션을 구현.
5. 뒤끝 서버 및 유니티 연동
    - 플레이어의 정보와 점수를 저장하고 열람할 수 있음.
